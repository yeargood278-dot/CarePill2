# CarePill2
# 1安服---药安服APP
1. 开发的硬件环境
处理器 i5/i7 或同级别芯片，内存 8GB 以上，可用硬盘空间 500GB 以上的个人计算机。

2. 运行的硬件环境
具备震动马达、音频输出设备（扬声器）、摄像头及 2GB 以上可用运行内存的安卓或 iOS 智能手机。

3. 开发该软件的操作系统
Windows 10/11 专业版或 macOS 12.0 及以上版本。

4. 软件开发环境 / 开发工具
HBuilderX 4.0 以上版本，配合 Chrome 浏览器调试工具及手机模拟器（如 MuMu 模拟器）。

5. 该软件的运行平台 / 操作系统
Android 5.0 (API Level 21) 以上或 iOS 10.0 以上移动操作系统。

6. 软件运行支撑环境 / 支持软件
Uni-app 运行框架（DCloud Runtime），支持内置系统 WebView 或 Crosswalk 浏览器内核。

7. 编程语言
Vue.js 2.x、JavaScript (ES6+)、HTML5、CSS3 及微信小程序/App 原生 API 扩展。

8. 源程序量
约 1,800 - 2,800 行（包含逻辑脚本、样式代码、界面模板及配置文件）。

9. 开发目的
通过数字化手段管理复杂用药计划，利用多重警报防止漏服，并通过库存监控与报告分析确保用药安全有效。

10. 面向领域 / 行业
移动健康医疗、养老监护、慢性病患者自我管理及医药数字化服务行业。

11. 软件的主要功能（约 200 字）

本软件提供全方位的智能服药监护方案。核心功能包含：1. 计划录入：支持手动或快捷扫码录入药名、剂量、餐时要求及多时间点设置，默认自动生成一个月服药周期。2. 智能提醒：在设定时间触发强震动、视觉弹窗及原生 TTS 语音播报（如“该服降压药了”），确保高龄群体亦能感知。3. 库存追踪：系统依据打卡记录自动扣减库存，并对低于安全阈值的药品进行预警提示。4. 健康报告：自动生成全周期依从性分析，统计已服与漏服次数，直观展示历史用药趋势。5. 后台监护：实时监测服务运行状态，确保警报系统稳定触发。

12. 软件的技术特点（约 100 字）
该软件基于 Uni-app 跨平台技术，实现“一套代码，多端运行”。核心技术亮点在于：集成了底层原生 TTS 语音引擎与长震动接口，打造多维强力警报；采用本地化持久存储技术，在离线状态下依然能保障数据安全与隐私；通过精准的时区适配算法，解决了 UTC 时间偏差导致的提醒失效问题。
